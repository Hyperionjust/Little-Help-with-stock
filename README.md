# stock-metrics-pro

**一个开源的个股基本面与技术面分析工具。** 输入股票代码，自动完成数据采集、指标计算与质量校验，输出四种格式的分析报告。覆盖 A股、港股、美股；对医药与生物科技公司，额外提供研发管线估值（rNPV）、临床进展与专利到期分析。

本工具面向金融行业从业者设计，**不要求编程背景**：既可作为 Claude 的插件通过对话使用，也可按本文档的分步指引在本地运行。

> **重要声明**：本工具仅执行数据整理与计算，输出不构成投资建议，不包含任何买卖推荐或价格预测。

## 1. 工具概述

### 1.1 解决什么问题

分析一只股票通常需要在多个工具之间切换：行情终端查K线，财报数据库查盈利指标，再用 Excel 手工搭建估值底稿。本工具将这一流程整合为单一指令：

```
输入：  分析 600519
输出：  网页版分析仪表盘 + Excel 底稿 + 文字摘要 + 结构化数据文件
```

### 1.2 覆盖范围

| 维度 | 内容 |
|---|---|
| 市场 | A股（沪深、科创板、创业板）、港股、美股 |
| 估值指标 | PE（TTM/静态/预测）、PB、PS、PEG、EV/EBITDA、股息率、市值 |
| 盈利质量 | ROE（含杜邦三因素分解）、ROA、ROIC、毛利率、净利率、利润现金含量 |
| 财务预警 | 应收账款增速与营收增速的背离、存货积压信号、利润与经营现金流的背离度 |
| 偿债与营运 | 资产负债率、流动比率、速动比率、利息保障倍数、应收/存货周转天数 |
| 成长性 | 营收与净利润的同比、环比、3年/5年复合增长率 |
| 技术面 | 均线系统（MA5–MA250）、MACD、RSI、KDJ、乖离率、波动率、量比、相对基准指数的超额收益、支撑与压力位 |
| 医药专项 | 管线 rNPV 估值、临床试验看板、专利到期（LOE）分析、催化剂日历 |

### 1.3 代码识别

支持多种输入格式，无需记忆规范写法：`600519`、`sh600519`、`600519.SH`、`0700.HK`、`AAPL`，以及公司中文名（如"贵州茅台"、"腾讯"）。

## 2. 核心设计：三项可靠性机制

通用 AI 工具直接回答财务数据问题时，存在一个已知风险：模型可能对网页上的数字进行"心算"，得出错误结果且表述自信。本工具通过三项机制在设计层面控制该风险。

**机制一：计算与解读分离。**
报告中的所有数值均由 Python 程序计算产生；AI 仅负责将计算结果转述为文字。工具内置校验程序，将解读文字中出现的每个数字与计算结果逐一比对，不一致即报错拦截。

**机制二：全量数据溯源。**
每个关键数值附带三项元信息：数据来源、采集时间、计算公式。在网页仪表盘中，将鼠标悬停于任意数字即可查看。医药模块的每项估值假设额外标注可信度等级：来自数据库或公告的事实、行业基准值、或需人工确认的主观假设。

**机制三：输出前质量校验。**
报告生成前自动执行检查，包括：核心数据是否缺失、两个独立来源的股价是否一致（偏差超过 1% 即拦截）、财报是否过期（超过 12 个月即拦截）。未通过检查时，工具输出带有明确警告标识的降级报告，而非隐瞒数据缺陷。

## 3. 输出说明

每次分析产出四种格式，数值完全一致（均由同一份计算结果生成，并经程序自动对账）：

| 输出 | 文件 | 用途 |
|---|---|---|
| 网页仪表盘 | `600519_dashboard.html` | 主要阅读界面。深色主题，含K线图、指标卡片、质量校验面板；双击打开，无需联网 |
| Excel 底稿 | `600519_workbook.xlsx` | 二次分析工具。原始数据与比率分离，比率单元格为公式（非数值），修改原始数据即自动重算 |
| 文字摘要 | 对话内呈现 | 核心指标表与要点解读 |
| 数据文件 | `600519_analysis.json` | 结构化完整数据，供程序化调用；一般使用者无需关注 |

报告样例见本仓库 `examples/` 文件夹（贵州茅台、恒瑞医药、Apple 三份真实数据仪表盘，下载后双击打开）。

## 5. 安装与使用：方式二（本地独立运行）

适用对象：希望在自己电脑上直接运行的使用者。共五个步骤，其中前两步仅首次需要。每一步只需按说明复制、粘贴、回车。

### 第 1 步：安装 Python（仅首次）

Python 是运行本工具所需的免费基础软件。

**Windows：**

1. 访问 [python.org/downloads](https://www.python.org/downloads/)，点击页面上方的下载按钮
2. 运行安装程序。**注意：安装界面底部有一个复选框 "Add Python to PATH"，必须勾选**——未勾选是后续报错的最常见原因
3. 点击 Install Now，等待完成

**Mac：**

1. 访问 [python.org/downloads](https://www.python.org/downloads/)，下载并运行安装包，按默认选项完成

**验证安装：**

- Windows：按 `Win + R`，输入 `cmd`，回车，在弹出的命令行窗口中输入 `python --version` 并回车
- Mac：按 `Command + 空格`，输入"终端"并回车，在窗口中输入 `python3 --version` 并回车

显示 `Python 3.x.x` 版本号即为安装成功。

### 第 2 步：下载本项目（仅首次）

1. 在本项目 GitHub 首页，点击绿色 **`< > Code`** 按钮，选择 **Download ZIP**
2. 解压至任意位置（如桌面），得到 `stock-metrics-pro` 文件夹

### 第 3 步：打开命令行并定位到项目文件夹

**Windows：** 打开解压后的 `stock-metrics-pro` 文件夹，点击窗口上方的地址栏，输入 `cmd` 并回车。命令行窗口将直接在该文件夹位置打开。

**Mac：** 打开"终端"，输入 `cd `（注意 cd 后有一个空格），将 `stock-metrics-pro` 文件夹拖入终端窗口，回车。

### 第 4 步：安装依赖组件（仅首次）

在命令行窗口中粘贴以下内容并回车：

```
pip install -r requirements.txt
```

（Mac 用户如提示找不到 pip，请改用 `pip3 install -r requirements.txt`。）

屏幕将滚动显示下载进度，约一至两分钟完成。结尾无红色错误信息即为成功。

### 第 5 步：生成第一份报告

在命令行窗口中依次执行以下两行（每行粘贴后回车）：

```
cd scripts
python run_analysis.py "分析 600519"
```

数据采集与计算约需 30 秒至 1 分钟。完成后，在 `scripts` 文件夹中双击打开 `600519_dashboard.html` 即可查看报告。

更换分析对象，只需修改引号内的内容：

```
python run_analysis.py "AAPL"
python run_analysis.py "看一下 00700 腾讯"
python run_analysis.py "分析 恒瑞医药" --outdir 输出文件夹
```

### 常见报错对照

| 报错信息 | 原因 | 解决方法 |
|---|---|---|
| `'python' 不是内部或外部命令` / `command not found` | 安装 Python 时未勾选 "Add Python to PATH" | 重新安装 Python，勾选该选项 |
| `'pip' 不是内部或外部命令` | 同上 | 同上，或改用 `python -m pip install -r requirements.txt` |
| `No such file or directory: run_analysis.py` | 命令行当前位置不在 scripts 文件夹 | 先执行 `cd scripts`，再运行分析命令 |
| 大量 `Timeout` / `ProxyError` | 所在网络（常见于公司内网）限制了数据源访问 | 参见[第 8 节](#8-数据来源与网络配置) |
| 报告顶部出现红色警告 | 属于正常功能：质量校验发现数据问题并如实提示 | 按警告文字判断具体数据缺陷 |

## 6. 报告阅读指引

建议按以下顺序阅读网页仪表盘：

**第一，查看顶部的质量校验面板。** 这是判断本份报告可信度的前提：绿色表示数据检查全部通过；黄色表示存在提示项（如个别字段缺失），报告可用但需留意；红色表示存在严重数据缺陷（如财报过期、股价缺失），报告已降级，数值仅供参考。

**第二，查看指标卡片。** 鼠标悬停于任意数值，可查看其数据来源与计算公式。

**第三，查看财务预警信号灯。** 该区域比对应收账款、存货与营收的相对增速，以及利润与经营现金流的背离程度，用于识别"账面利润良好、但现金回流恶化"的情形。红色信号值得进一步核查。

**第四，如需自行调整假设，使用 Excel 底稿。** Ratios 工作表中的比率均为公式（引用 Raw Data 工作表），修改原始数据后自动重算；医药标的另有 rNPV 工作表，估值假设为可编辑单元格。

## 7. 医药模块说明

医药行业的估值逻辑与一般行业不同：一家当前亏损的生物科技公司，其价值可能主要来自尚未上市的在研药物。因此，当识别到医药或生物科技标的时，工具在通用分析之外自动增加以下内容。

**研发管线估值（rNPV，风险调整净现值）。** 计算逻辑为：对每个在研药物，估算其上市后的销售规模，乘以从当前研发阶段走到最终获批的累计成功概率，减去尚需投入的研发成本，再折算为现值。逐药计算后加总，并计入公司账面净现金，得到公司层面的管线估值。成功概率取自行业公开统计（不同研发阶段、不同治疗领域各有基准值），并逐项披露。

**临床试验看板。** 数据取自 ClinicalTrials.gov（美国临床试验注册库），按药物、适应症、研发阶段、试验状态结构化呈现，并单独标出已终止的试验。

**专利到期分析。** 列示已上市药物的专利到期时间。若某药物贡献公司收入 20% 以上且专利在 5 年内到期，将被标记为高风险项。

**需人工核对清单。** rNPV 计算中所有主观假设（峰值销售、市场渗透率、专利到期年份等）汇总为一张独立清单，逐项列明取值、假设性质、以及该假设变动对估值结果的影响方向。

> **使用提示**：自动生成的 rNPV 基于行业平均假设，用于判断量级，不应作为目标价。进行严肃估值时，请在 Excel 底稿的 rNPV 工作表中将关键假设替换为自己的判断，结果将实时重算。

## 8. 数据来源与网络配置

### 8.1 数据来源

全部数据取自免费公开接口，无需注册或付费：

| 数据 | 来源 |
|---|---|
| A股、港股行情 | 腾讯财经、新浪财经 |
| A股财务报表 | 东方财富 |
| 美股、港股行情与财报 | 雅虎财经 |
| 临床试验数据 | ClinicalTrials.gov（美国官方注册库） |
| 药品批准信息 | openFDA（美国药监局公开接口） |

关键数据（如股价）同时从两个独立来源获取并交叉核对；单一来源失效时自动切换备用来源。数据无法获取时，工具在报告中明确标注缺口，不以估计值填充。

### 8.2 受限网络环境的配置

在企业内网等设有访问白名单的网络环境中运行时，可能出现大量连接超时。解决方法是请网络管理员将以下域名加入放行清单（均为正规公开数据接口）：

```

  web.ifzq.gtimg.cn              腾讯K线
  hq.sinajs.cn                   新浪行情
  datacenter-web.eastmoney.com   东方财富财报
  datacenter.eastmoney.com

美股、港股及指数必需：
  query1.finance.yahoo.com       雅虎财经
  query2.finance.yahoo.com

医药模块需要：
  clinicaltrials.gov             临床试验
  api.fda.gov                    药品批准

备用来源（可选，提高稳定性）：
  push2his.eastmoney.com
  push2.eastmoney.com
  33.push2his.eastmoney.com
  basic.10jqka.com.cn
```

如网络限制无法解除，可采用第 9 节的本地数据方案，实现完全离线运行。

## 9. 使用自有数据（同花顺 iFind / Wind 导出）

付费终端的财务数据在完整性与准确性上优于免费来源。工具支持直接读取从 iFind、Wind 等终端导出的 Excel 或 CSV 文件：中文字段名（如"营业总收入"、"归属于母公司股东的净利润"）自动识别，金额单位（元、万元、亿元）自动换算。

**推荐用法——自有财报数据与在线行情结合：**

```
python import_local.py --symbol 600519 --financials 导出的财报.xlsx --unit 元 --complement-live -o raw.json
python run_analysis.py 600519 --offline-fixture raw.json
```

第一行将导出文件转换为工具的标准数据格式：财务数据以导入文件为准，`--complement-live` 参数指示工具从免费来源补充股价与K线。第二行基于该数据生成报告。

**完全离线用法：** 将K线数据一并导出为 CSV，追加 `--kline 导出的K线.csv` 参数，全程无需联网。

**字段扩展：** 如导出文件的字段命名不在内置识别范围内，可在 `scripts/import_local.py` 文件头部的 `FIELD_ALIASES` 对照表中自行补充，格式参照现有条目。

## 10. 常见问题

**是否收费？**
不收费。数据来源均为免费公开接口，工具本身开源。

**是否提供买卖建议？**
不提供。本工具仅输出数据整理与计算结果。

**数据准确性如何？**
免费来源的数据在绝大多数情况下与主流行情终端一致，工具另有跨来源交叉核对机制。但免费接口可能存在延迟或缺失；缺失项在报告中明确标注。正式用途请以交易所披露及公司公告为准。

**支持哪些标的？**
A股（含科创板、创业板）、港股、美股的上市公司。

**医药 rNPV 结果的可靠性？**
计算方法为行业通行的风险调整净现值法，但任何 rNPV 结论均取决于输入假设。工具将全部假设集中披露于"需人工核对清单"，供使用者自行校准。建议将自动结果视为量级参考，重要决策前替换为自己的假设。

**单次分析耗时？**
在线模式约 30 秒至 1 分钟；本地数据模式更快。

**支持的操作系统？**
Windows、macOS、Linux 均可运行。
