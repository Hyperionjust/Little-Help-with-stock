# 风险框架

> **何时读这份文件**：当你要把"风险提示"从几条零散 bullet 升级成可打分、可排序、可与估值通约的结构化评估时读它。深度研报必读。

---

## 为什么散点式风险提示没用

大多数报告的风险段是这样的："宏观波动风险、行业竞争风险、政策风险"——正确、全面、且毫无用处。因为它没告诉读者：哪个风险更可能发生？发生了会亏多少？怎么提前知道它要来了？

好的风险框架做三件事：**给每条风险打概率、把影响锚定到估值冲击、配一个监控信号**。这样风险就从"提醒你小心"变成了"可以纳入决策的定量输入"。

**规模要求**：识别 **8-16 条**互相独立的风险，覆盖经营、行业、外部三个层次。太少不全面，太多则失焦。

---

## 三个层次的风险，缺一不可

从公司内部一路铺到外部环境，避免只盯着经营执行而漏掉系统性冲击。

**层次一：内生经营与治理（4-7 条）**
- 战略执行（新品交付、产能爬坡、并购整合落地）
- 组织与关键人（核心团队流失、创始人/CEO 继任）
- 客户与供应商集中度（大客户依赖、单一供应绑定）
- 技术与研发（路线押错、研发失败、IP 被挑战）
- 资本配置（过度扩张、并购商誉、分红与回购取舍）
- 治理与信息质量（审计、关联交易、披露透明度）

**层次二：竞争格局与需求（2-4 条）**
- 竞争加剧（新进入者、价格战、份额流失）
- 需求波动（周期敏感、渗透见顶、换代节奏）
- 模式或技术被颠覆（替代技术、监管重塑规则）
- 上游供应与成本（原料涨价、单一来源、地缘扰动）

**层次三：财务、估值与外部环境（3-5 条）**
- 杠杆与再融资（债务/EBITDA、利息保障、到期墙）
- 流动性与现金流（现金跑道、营运资金、自由现金流缺口）
- 利率与汇率（资本成本、久期、币种错配）
- 估值层面（倍数压缩、DCF 敏感、拥挤交易）
- 政策与监管（反垄断、数据合规、环保、行业专属）
- 地缘与宏观（贸易摩擦、制裁、衰退、通胀）

## 打分：0-1 连续标尺，影响锚定到估值冲击

**这是本框架与常规风险表最大的不同**。不用"高/中/低"这种粗桶，而用 **0 到 1 的连续概率**，并把**影响直接锚定成"对内在价值的百分比冲击"**。

**为什么要这么做**：只有当影响用"对估值的 % 冲击"来表达时，风险才和 DCF **可通约**——你才能把一条 0.3 概率、-20% 影响的风险，和 DCF 里 WACC 上移 50bp 的效果放在同一把尺子上比较、甚至加权求和。用"高/中/低"永远做不到这种通约。

**概率**：0.0-1.0 的连续值。
- 参考锚点：0.10-0.25 偏低、0.25-0.50 中等、0.50-0.75 较高、>0.75 很高。
- 锚点只是参考，鼓励给出更精细的值（如 0.35）。写出你为什么定这个数。

**影响**：对内在价值的百分比冲击（正负号表方向，通常为负）。
- 参考锚点：±5% 以内偏小、±5-15% 中等、±15-30% 较大、>±30% 极大。
- 影响要落到具体机制：这条风险是砸收入、砸利润率、还是砸估值倍数？各砸多少个百分点，最后合成多少 % 的估值冲击。

**优先级 = 概率 × |影响|**。这就是这条风险的期望估值损失（%），是排序的唯一依据。优先级最高的几条必须进摘要。

## 逐条风险数据卡

每条风险（8-16 条）都填：

| 字段 | 内容 |
|---|---|
| **风险 ID** | R1, R2, R3… |
| **层次** | I 经营 / II 竞争 / III 外部 |
| **风险标题** | 5-10 字简称 |
| **描述** | 会出什么问题？（50-80 字） |
| **概率** | 0.0-1.0 连续值 + 定这个值的理由 |
| **影响** | 对估值的 % 冲击 + 传导机制（砸收入/利润率/倍数各多少） |
| **优先级** | 概率 × \|影响\| = 期望估值损失% |
| **时间尺度** | 近期(0-12月) / 中期(1-3年) / 长期(3年+) |
| **缓释** | 公司在做什么应对？（或：未缓释） |
| **监控信号** | 哪个数据点会提示这条风险正在兑现 |
| **关联情景** | 主要影响乐观/中性/悲观哪个情景 |

## 风险叙述

**Top 3 风险——展开讨论**（每条 150-200 字）：
1. 风险机制的完整描述
2. 历史先例（这家公司或同业出现过吗？）
3. 量化影响（收入、盈利、估值会怎样，落到具体 %）
4. 管理层应对（在做什么、能做什么）
5. 我方评估（概率加权后的看法，市场定价对了吗？）

**其余风险——紧凑格式**（每条 80-120 字）：风险名 + 描述 + 概率判断 + 监控信号。

## 风险-收益综合（收尾 150-200 字）

- 整体风险画像：高风险高回报，还是低风险中回报？
- 风险不对称性：上行意外与下行风险是否平衡？把所有正向"风险"（超预期）也算进来。
- 风险定价：主要风险是否已反映在当前估值里？
- 对主线最致命的单一风险：哪一条最可能颠覆你的中性判断？

这段是风险框架的"所以呢"——把一堆分数收敛成一个对赔率的判断。

---

## 影响 % 怎么拆：一个算例

"这条风险影响 -25%"不能拍脑袋，要走一遍传导链。以"反垄断裁定"为例：

- **砸收入**：若被迫开放某项独家分成，相关业务线收入下滑约 8%，该业务占总收入 40% → 总收入 -3.2%。
- **砸利润率**：合规成本上升，营业利润率压缩约 1.5pp。
- **砸倍数**：监管不确定性抬升风险溢价，市场给的 PE 从 20 倍降到 17 倍 → 倍数 -15%。
- **合成**：收入与利润率变化经 DCF 传导约 -10% 内在价值，叠加倍数重估，综合估值冲击约 -25%。

把这条链写出来，"-25%"就从一个凭感觉的数，变成一个可以被质疑、被复核、被接到 DCF 的结构化判断。概率同理——0.35 要能说出"参考同类案件的裁定倾向 + 当前调查进展"。

## 与情景框架的对接

风险和情景是同一枚硬币的两面：**悲观情景的触发事件，就是优先级最高的几条风险**。二者必须互相咬合——
- 每条 top 风险都应能在情景转换图里找到对应的"中性→悲观"转移触发器。
- 风险的"影响 %"应当能解释悲观情景相对中性的目标价跌幅。
- 若某条高优先级风险在情景框架里找不到落点，说明两个模块之一有遗漏，回去补。

## 质检清单

```markdown
□ 8-16 条独立风险，三层齐全（I:4-7, II:2-4, III:3-5）
□ 每条概率为 0-1 连续值，写明定值理由
□ 每条影响锚定到"对估值的 % 冲击"，并给出传导机制
□ 优先级 = 概率×|影响|，作为唯一排序依据
□ 每条配监控信号
□ Top 3 展开叙述（150-200字），其余紧凑（80-120字）
□ 风险-收益综合段，点明对主线最致命的单一风险
□ 每条 top 风险与情景转换图的"中性→悲观"触发器对齐
```
