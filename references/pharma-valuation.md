## 1. 范式判定

按 sum-of-the-parts 思路自动选择（`decide_paradigm()`）：

| paradigm | 触发条件 | 主体估值 | 叠加 |
|---|---|---|---|
| `commercial` | 有已上市重磅药、稳定收入（营收 > 0 且净利 > 0 或营收规模大） | 通用 EV/EBITDA、DCF | 专利悬崖调整 |
| `clinical_biotech` | pre-revenue 或收入极少、价值主要在管线 | **rNPV** | 管线看板 |
| `large_pharma_sotp` | 大型 pharma：既有上市组合又有实质在研管线 | 已上市组合常规估值 **+** 在研管线 rNPV | 二者相加，分部展示 |

脚本对 `large_pharma_sotp` 要能对同一公司同时算商业化部分与管线 rNPV 并分部展示。判定用营收/净利规模 +
管线中处于 III 期及以后的资产数作为信号；边界模糊时优先判 `large_pharma_sotp`（更保守、信息更全）。

## 2. rNPV 六步计算引擎

对单个管线资产（`compute_asset_rnpv()`）：

1. **峰值销售估算**：`目标患者数 × 渗透率 × 单患者年费用 × 疗程系数`（见 §5）。
2. **收入曲线建模**：上市 → 爬坡(默认 5 年线性到峰值) → 峰值平台 → LOE 后衰减（见 §7 衰减曲线）。
3. **PoS 逐阶段赋值 → 累积 PoS**：当前阶段从 ClinicalTrials.gov 实读；`累积 PoS = 从当前阶段到批准各
   阶段成功率连乘`（见 §3）。TA（治疗领域）按适应症映射后取对应基准。
4. **概率加权现金流**：每期现金流 × 累积 PoS。
5. **折现**：用**独立的 `clinical_discount_rate`**（见 §4），**不复用通用 WACC**。
6. **减去剩余研发成本**（概率加权），得该资产 rNPV。

公司层：`company_rnpv = Σ asset_rnpv + net_cash − debt`。输出必须能分资产、分阶段拆解（哪个药、哪个适应症、
哪个阶段贡献多少价值），不是黑箱单值。

## 3. PoS 阶段转换基准表

默认单阶段成功率（`PHASE_TRANSITION_POS`，`benchmark`，可覆盖）：

| 转换 | 默认成功率 |
|---|---|
| Preclinical → Phase 1 | 0.45 |
| Phase 1 → Phase 2 | 0.60 |
| Phase 2 → Phase 3 | 0.30 |
| Phase 3 → NDA/BLA | 0.55 |
| NDA/BLA → Approval | 0.88 |

累积（从各阶段到批准，连乘校验用）：
- 从 Phase 1 到批准 ≈ `0.60 × 0.30 × 0.55 × 0.88 ≈ 0.087`（落在行业 7–12% 区间）。
- 从 Phase 2 到批准 ≈ `0.30 × 0.55 × 0.88 ≈ 0.145`。
- 从 Phase 3 到批准 ≈ `0.55 × 0.88 ≈ 0.484`。
- 从 NDA 到批准 ≈ `0.88`。

**治疗领域调整**（`TA_POS_MULTIPLIER`，作用于累积 PoS 或分阶段，脚本对累积做乘子并夹取到合理上下限）：
- 肿瘤 Oncology：整体偏低，从 P1 累积 ≈ 5–8%（乘子 ~0.7）
- 罕见病 Rare disease：偏高，从 P1 累积 ≈ 15–25%（乘子 ~1.8）
- 其它领域：乘子 1.0

映射：适应症字符串 → TA（`map_indication_to_ta()`，关键词匹配，未命中归 `other`）。

**门禁相关**：对临床前/早期（当前阶段 ≤ Phase 1）资产，若算出的累积 PoS > 30%，判 critical——几乎必然是阶段
判定错误或连乘写反（例如把"单阶段"当"累积"）。

## 4. 折现率规则——不双重计罚

**这是医药模块最容易犯的错，单列强调。**

rNPV 的折现率**必须低于**常规 DCF 的 WACC。原因：临床失败风险已经通过 PoS 概率加权**单独扣除**了；如果再用
含高风险溢价的 WACC 折现，等于对同一个临床风险罚了两次（double counting / double penalty），会系统性低估管线
价值，把本该有价值的早期资产算成接近零。

规则：
- 脚本用**独立参数 `clinical_discount_rate`**，与通用 `wacc` 是两个变量，`pharma_valuation.py` 里**禁止**把
  `wacc` 传给 rNPV 折现。
- 分层默认（`CLINICAL_DISCOUNT_RATE_BY_PARADIGM`，`benchmark`，可覆盖）：
  - `large_pharma_sotp` 管线部分：**10%**
  - `clinical_biotech`：**12.5%**
  - pre-revenue 早期（无 III 期资产的 clinical_biotech）：**15%**
  - 允许区间 8–20%（早期偏高端），但仍显著低于典型 biotech WACC。
- 门禁：`quality_gate` 检测 `pharma.double_penalty_check.passed`——若 `clinical_discount_rate` 等于
  `wacc_general`（或脚本标记未使用独立参数），判 **critical** 并阻断。

## 5. 峰值销售估算

`峰值销售 = 目标患者数 × 渗透率 × 单患者年费用 × 疗程系数`

- **目标患者数**：优先 ClinicalTrials.gov 适应症 + 公开流行病学（若环境有 `rare-disease-epi` skill 可交叉
  参考，但本 skill 自洽取数）。标 `hard` 或 `benchmark`。
- **渗透率**（默认 `PENETRATION_BY_COMPETITION`，**永远标 `user_assumption`**）——按竞争格局分层，脚本按
  ClinicalTrials.gov 同适应症在研试验密度粗判竞争度：
  - first-in-class / 无直接竞品：**25%**
  - 2–3 个竞品：**15%**
  - 拥挤赛道（≥4 竞品）：**8%**
- **单患者年费用、疗程系数**：默认走行业基准或用户输入，标 `user_assumption`（定价主观性极强）。

## 6. 敏感性分析

对每个资产及公司层，对四个变量做单因素敏感性（tornado 数据）：`PoS`、`峰值销售`、`折现率`、`上市时间`。
每个变量取 `low / base / high` 三档（默认 ±30% 幅度，折现率用 ±200bp），其余固定，重算 rNPV，输出区间。
展示区间而非单点——因为这些假设主观性强，给区间比给一个精确假数字更诚实。

## 7. 专利悬崖 / LOE

- **专利瀑布**：对已上市药按到期年份排列，估每药收入占比，识别"单药收入占比 >20% 且 5 年内到期"的高危悬崖。
- **衰减曲线分子型分开**（`LOE_DECAY`）：
  - 小分子（ANDA 仿制路径，衰减快）：LOE 后首年即失去 ~60–80% 销售，默认首年保留 30%、逐年再降。
  - 生物药（biosimilar 路径，衰减慢）：LOE 后逐步侵蚀，默认首年保留 80%、5 年降到 ~40%。
- openFDA 可查批准信息；**专利到期精确日期属付费源领域**，第一版用公开可得信息 + 标 `user_assumption` 的
  估计，并列入需人工核对清单。

## 8. 催化剂日历

从临床试验的 `primaryCompletionDate` + 公开可查的 PDUFA/审批节点，生成未来 12–24 个月催化剂时间线。
**读得到就填，读不到就在需人工核对清单标注，绝不编造日期。**

## 9. 需人工核对清单规则

所有 `source_type == "user_assumption"` 的假设**必须**逐项出现在 `pharma.human_verification_checklist`，
一个不漏（`test_pharma` 与 `quality_gate` 都会检查）。每项含：取值、来源类型、改动它对 rNPV 的影响方向
（如"↑渗透率 → ↑rNPV"）。这是医药估值诚实性的底线。

## 10. 玩具资产 golden case

用于锚定 rNPV 引擎正确性，闭式手算，参数固定、不依赖任何外部数据。定义（见 `tests/golden.json`
的 `pharma_toy_asset`）：

```
单管线资产，估值基准年 = 2026 (t=0)
现金流（未加权，单位：百万），发生在各年年末：
  t=1: -50   (剩余研发成本)
  t=2: +100
  t=3: +300
  t=4: +300
  t=5: +200
当前阶段 = Phase 3  →  累积 PoS = 0.55 × 0.88 = 0.484
clinical_discount_rate = 0.10
net_cash = 0, debt = 0
```

**闭式手算**（概率加权后折现；研发成本与收入统一用同一累积 PoS 加权——玩具资产刻意简化，真实脚本可对研发成本
与收入用不同处理，但玩具锚保持单一 PoS 以便闭式验证）：

```
每期加权现金流 = CF_t × 0.484
PV = Σ_t (CF_t × 0.484) / 1.10^t
   = 0.484 × [ -50/1.1^1 + 100/1.1^2 + 300/1.1^3 + 300/1.1^4 + 200/1.1^5 ]
折现因子: 1.1^1=1.10, 1.1^2=1.21, 1.1^3=1.331, 1.1^4=1.4641, 1.1^5=1.61051
括号内 = -45.4545 + 82.6446 + 225.3944 + 204.9040 + 124.2237 = 591.7122
asset_rnpv = 0.484 × 591.7122 = 286.389  (百万)
company_rnpv = 286.389 + 0 − 0 = 286.389
```

golden 值：`asset_rnpv ≈ 286.39`（容差 ±0.5%）。累积 PoS golden：`0.484`。
折现率独立性断言：`clinical_discount_rate (0.10) != wacc_general`。
