# 数据源接入说明

这份文档讲清楚两件事:**哪些数据源开箱即用**(不用配置),以及**怎么接入你自己的付费终端**(iFinD、Wind、Tushare Pro、标普、Bloomberg 等)。

---

## 一、开箱即用的免费源(无需任何配置)

装好工具就能用,**不需要任何 API key,不需要接入任何东西**。这些源已内置在降级链里,自动按可靠性排序、自动跨源校验:

| 市场 | 免费源(按优先级) | 覆盖 |
|---|---|---|
| A股 | 腾讯 → 新浪 → akshare(东财) → efinance → baostock | 行情、K线、年报三表 |
| 港股 | 腾讯 → 新浪 → Yahoo → akshare → efinance | 行情、K线、财报 |
| 美股 | Yahoo → 腾讯 | 行情、K线、完整年报三表 |
| 医药临床 | ClinicalTrials.gov + openFDA | 管线阶段、批准信息(全球,免费) |

**它们能支撑什么**:指标看板、投资速览、以及美股的深度研报,免费源都够用。

**它们的短板**(这也正是付费源的价值所在):

- **季度财报**:免费源普遍只稳定提供年报。A股季报时有时无,港股多为半年报。
- **一致预期**:免费源基本拿不到分析师预测,所以 `pe_forward`(预测市盈率)经常是空的。
- **分部收入**:按业务线拆分的营收,免费源几乎没有。
- **可比公司全量倍数**:能取到,但字段偶有缺失。

所以:**A股/港股的深度研报(三表建模+DCF)在纯免费源下会因季度数据缺失而降级**。工具会主动降级并在报告里声明,不会硬做。想要完整精度,就接一个付费源。

---

## 二、接入付费终端

有任意付费终端的凭证,放进配置即自动升为**Tier-0**(降级链最高优先级),并解锁上面那些免费源拿不到的字段。

> **一条纪律**:付费不等于免信任。付费源升 Tier-0,但价格和市值仍会跟免费源交叉校验——差超 1% 照样拦。这道防线不因数据源付费而关闭。

### 方式速查

| 你手上有的 | 用哪种方式 | 难度 |
|---|---|---|
| Tushare Pro 的 token | API token(内置适配器,填个环境变量) | ⭐ 最简单 |
| Wind/iFinD 导出的财报 Excel | 导出文件(直接把文件发给工具) | ⭐ 最简单 |
| iFinD 的 API 凭证 | API token(内置适配器,可能需核对字段) | ⭐⭐ |
| Wind 的 Python 接口(本地装了 WindPy) | 本地客户端 | ⭐⭐⭐ |
| 标普 S&P Capital IQ 的 API | 自定义适配器(照模板填) | ⭐⭐⭐ |
| Bloomberg 终端 | 本地客户端(需桥接脚本) | ⭐⭐⭐⭐ |

---

### Tushare Pro(最简单,推荐先用这个)

Tushare Pro 是国内最易得的付费 A股数据源,注册即送积分。

1. 在 [tushare.pro](https://tushare.pro) 注册,拿到你的 token
2. 设一个环境变量:

   **Windows(命令提示符):**
   ```
   setx TUSHARE_TOKEN 你的token字符串
   ```
   设完**重开一个命令行窗口**才生效。

   **Mac/Linux:**
   ```bash
   export TUSHARE_TOKEN=你的token字符串
   ```
   想永久生效,把这行加到 `~/.zshrc` 或 `~/.bashrc`。

3. 验证接通了:
   ```
   python engine/providers/paid/registry.py
   ```
   看到 "可用: Tushare Pro" 就成了。之后所有 A股分析自动优先用它。

内置适配器 `tushare_pro.yaml` 已备好,不用改任何东西。

---

### iFinD(同花顺)

iFinD 有两种接入形态,看你拿到的是哪种:

**A. 你有 iFinD 的 API token**

设环境变量,内置适配器 `ifind.yaml` 会自动生效:
```
setx IFIND_TOKEN 你的token          （Windows）
export IFIND_TOKEN=你的token         （Mac/Linux）
```

⚠️ **首次接入务必核对**:iFinD 的接口字段命名偶有调整,内置适配器是按公开文档写的。第一次用时,拿一只你熟悉的股票跑一遍,核对市值、营收、净利润这三个数跟同花顺终端界面一致,再放心用。若对不上,打开 `engine/providers/paid/adapters/ifind.yaml`,按里面的注释调整 `path`(上游字段名)。

**B. iFinD 是写文件型接口(结果落 CSV)**

同花顺很多接口是"调用后把结果写成 CSV 文件"而非直接返回。如果你的是这种,更简单——直接用**导出文件方式**(见下),把 iFinD 生成的 CSV 发给工具即可,连 token 都不用配。

---

### Wind(万得)

**最省事的方式:导出文件。** 不需要 API,不需要 WindPy。

在 Wind 终端里把公司的三张财务报表导出成 Excel(利润表/资产负债表/现金流量表),然后:

- 要么在对话里直接把文件发给工具,说"用这份 Wind 导出重算"
- 要么设 `ERS_IMPORT_FILE` 环境变量指向文件路径

内置的 `wind_export.yaml` 适配器用中英别名匹配表头,Wind 中英文界面导出的都能吃,两种常见布局(每行一个期间 / 科目为行期间为列)都自动识别。

**如果你本地装了 WindPy(Wind 的 Python 接口):** 这属于"本地客户端"方式,需要一个桥接脚本把 WindPy 查询封装成命令行。这个稍复杂,需要的话我可以单独帮你写。

---

### 标普 S&P Capital IQ / Market Intelligence

标普没有内置适配器(它的接口凭证形态因订阅而异),用**自定义适配器**照模板填:

1. 复制模板:
   ```
   cp engine/providers/paid/adapters/custom.template.yaml \
      engine/providers/paid/adapters/sp_capiq.yaml
   ```

2. 编辑 `sp_capiq.yaml`,主要填四处:
   - `auth.env_var`:你把 S&P 的 API key 放进哪个环境变量(如 `SP_API_KEY`)
   - `transport.endpoints`:S&P 数据接口的请求 URL(查你的 S&P API 文档,`{symbol}` 和 `{token}` 会自动替换)
   - `capabilities`:标普强在一致预期和标准化财务,建议开 `[quote, financials, estimates]`
   - `fields`:标普字段名 → 本工具标准字段名,**注意填对单位**(标普美股数据多为百万美元计,填 `million`)

3. 验证:`python engine/providers/paid/registry.py`

模板里每个字段都有注释,照着填即可。填完拿一只美股核对一下数值。

> 说明:本工具不含标普的任何代码或数据。你需要自备有效的 S&P 订阅与 API 凭证——这份说明只告诉你如何把你自己的凭证接进来。

### Bloomberg 终端

Bloomberg 数据必须经本地 Terminal(blpapi),无法用简单的 token 接入,最复杂。

内置的 `bloomberg.yaml` 已备好字段映射(用 Bloomberg 的标准字段名如 `PX_LAST`、`CUR_MKT_CAP`),但需要一个把 blpapi 查询封装成命令行的桥接脚本,把它的路径放进 `BLOOMBERG_BRIDGE` 环境变量。

桥接脚本需要在装了 blpapi 的机器上写,这一步需要的话单独找我。

---

### 接一个内置列表里没有的终端

复制 `custom.template.yaml`,按注释填,放回 `engine/providers/paid/adapters/` 目录,**不用改任何代码**。模板里讲得很细,核心就四块:key 放哪、请求 URL 怎么写、能提供哪些数据、字段和单位怎么对应。

## 三、单位:唯一容易填错的地方

自定义适配器时,`fields` 里每个字段要声明**原始单位**。这是唯一需要小心的地方。

工具靠这个把 元/万/亿/百万 统一成内部口径。举个真实的坑:如果把"元"计价的市值标成了"百万",市值会被当成小 100 万倍,PE/PB/PS 全错。

好在有**量纲哨兵**兜底:市值 ÷ 价格得出的隐含股本如果跟申报股本差超 1000 倍,工具直接报错拒绝出报告。但最好一开始就填对。

单位可选值:`native`(每股价格类,不换算)、`元`、`千`、`万`、`百万`、`亿`,或英文 `thousand`/`million`/`billion`。完整清单在 `engine/_util.py` 的 `UNIT_FACTORS`。

## 四、接入后怎么确认生效了

- 跑 `python engine/providers/paid/registry.py`,你的终端出现在"可用"列表里
- 出的报告首页会标明数据层级:`data_tier: 0` 表示用了付费源,并注明是哪个
- `pe_forward`(预测市盈率)、分部收入这些免费源拿不到的字段开始有值了
